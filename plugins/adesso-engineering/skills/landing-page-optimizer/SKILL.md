# Landing Page Optimizer

Plan and build high-converting landing pages in Drupal/adesso CMS.

## The Formula for High-Converting Landing Pages

This skill follows the proven "High Converting Landing Page Formula" with research-backed patterns:

```
┌─────────────────────────────────────────────────────────────┐
│ 1. HERO (Attention)                                         │
│    Eyebrow + Headline (3 outcomes) + CTA + CTA Subtext     │
│    + Product Visual + Client Logos                          │
├─────────────────────────────────────────────────────────────┤
│ 2. VALUE SECTION (Interest)                                 │
│    Value-Driven Heading + 3-5 Core Outcomes + Icons         │
│    + Mini Proof Points + Micro CTA                          │
├─────────────────────────────────────────────────────────────┤
│ 3. OPTIONAL: Features / How It Works / Comparison / Team    │
├─────────────────────────────────────────────────────────────┤
│ 4. PRICING (if applicable)                                  │
│    Clear Plans + "Who it's for" + Most Popular + Social     │
├─────────────────────────────────────────────────────────────┤
│ 5. TESTIMONIALS (Social Proof)                              │
│    Stage-Setting Heading + Photos + Stats + Fresh Quotes    │
├─────────────────────────────────────────────────────────────┤
│ 6. FAQ (Objection Handling)                                 │
│    Categorized + Skimmable + Conversion Links               │
├─────────────────────────────────────────────────────────────┤
│ 7. FINAL CTA (Action)                                       │
│    Clear Headline + Benefit Subtext + Trust Boosters        │
├─────────────────────────────────────────────────────────────┤
│ 8. FOOTER (Trust & Navigation)                              │
│    Mini Sitemap + USP Tagline + Legal + Contact + Social    │
└─────────────────────────────────────────────────────────────┘
```

## Workflow

### 1. Analyze Input

Determine what you're working with:
- **Existing URL**: Fetch and analyze current structure, identify gaps
- **Topic/Service**: Research what content elements are needed
- **Competitor URL**: Extract best practices to apply

### 2. Identify Page Type & Audience

| Page Type | Primary Goal | Key Elements |
|-----------|--------------|--------------|
| Homepage | Orient & Route | Hero, Business Areas, Trust Signals, Contact |
| Service/Product | Convert | Hero+CTA, Benefits, Features, Social Proof, Form |
| Category | Navigate | Overview, Sub-categories, Key Benefits |
| Contact | Capture Lead | Multiple Contact Options, Form, Map |
| Pricing | Convert | Plans, Comparison, FAQ, Social Proof |

**Audience Detection**:
- B2B: Longer copy, detailed benefits, trust/certifications important
- B2C: Emotional triggers, shorter copy, urgency elements

## Section-by-Section Best Practices

### 1. HERO SECTION (Above the Fold)

The hero has 3-5 seconds to make a connection. Every element must work together:

```
┌────────────────────────────────────────────────────────────────┐
│  [Eyebrow Text]                    [Client Logos - Trust Bar]  │
│  "For modern remote teams"         Stripe | Notion | Salesforce│
│                                                                │
│  [H1: 3 Big Outcomes]                                         │
│  "Organize conversations, files, and tools in one place"       │
│                                                                │
│  [Primary CTA]  [CTA Subtext]                                 │
│  "Try it free"  "No credit card needed."                      │
│                                                                │
│  [Product Visual - Screenshot/Video/Dashboard]                 │
│  Shows instantly what the product does without scrolling       │
└────────────────────────────────────────────────────────────────┘
```

**Hero Checklist:**
- [ ] **Eyebrow text**: Sets context, clarifies who it's for (e.g., "Für Landwirte in NRW")
- [ ] **Headline (H1)**: Describes 3 big outcomes, not features (max 10 words)
- [ ] **CTA is benefit-focused**: "Jetzt Angebot anfordern" not "Absenden"
- [ ] **CTA Subtext**: Reduces hesitation ("Unverbindlich", "Kostenlos", "In 2 Minuten")
- [ ] **Product Visual**: Video or dashboard image helps users instantly understand
- [ ] **Client Logos**: Shows known clients → builds trust instantly

**Hero CTA Subtext Examples:**
| Subtext | Effect |
|---------|--------|
| "No credit card needed" | Removes payment fear |
| "Setup takes 2 minutes" | Reduces effort perception |
| "Unverbindlich anfragen" | Removes commitment fear |
| "Kostenlose Beratung" | Adds value |
| "Antwort in 24 Stunden" | Sets expectations |

### 2. VALUE SECTION ("Why Teams Love Using Us")

**DON'T**: "Our Features" / "What We Offer"
**DO**: "Why Teams Love Using Us" / "Warum Kunden uns vertrauen"

```
┌────────────────────────────────────────────────────────────────┐
│  [Value-Driven Heading]                                        │
│  "Why Teams Love Using Us"                                     │
│                                                                │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐            │
│  │   [Icon]    │  │   [Icon]    │  │   [Icon]    │            │
│  │ Work Faster │  │Stay Organized│  │Feel In Control│          │
│  │  Automate   │  │Keep everything│  │Less chaos     │          │
│  │  repetitive │  │  in one      │  │ More clarity  │          │
│  │   tasks     │  │   place      │  │  Every day    │          │
│  └─────────────┘  └─────────────┘  └─────────────┘            │
│                                                                │
│  [Mini Proof Point]                                           │
│  "Our onboarding dropped from 1 week to 2 days" – Sophia, Acme│
│                                                                │
│  [Micro CTA: "Start free trial"]                              │
└────────────────────────────────────────────────────────────────┘
```

**Value Section Checklist:**
- [ ] **Start with Value-Driven Heading**: Focus on outcomes, not features
- [ ] **3-5 Core Outcomes**: Not more (cognitive overload)
- [ ] **Simple icons**: Anchor attention, help users scan faster
- [ ] **Mini Proof Points or Quotes**: Real numbers/quotes add credibility
- [ ] **Finish with Micro CTA**: Don't make users scroll to take action

---

### 3. OPTIONAL SECTIONS

Include based on page goals:

| Section | When to Include | Purpose |
|---------|-----------------|---------|
| **Features** | Complex products | Help users understand exactly how product/service works |
| **How It Works / Process** | Service pages | Outline workflow or product journey step by step |
| **Comparison / Alternatives** | Competitive markets | Show why you're better than others |
| **Team / About Us** | B2B, trust-critical | Put faces behind the product, build credibility |

- [ ] **CTA Subtext**: "Switch plans or cancel anytime" removes fear
- [ ] **Social Proof below pricing**: Client logos, reviews, "Trusted by X"

### 6. FAQ SECTION

FAQs should drive conversion, not just answer questions:

```
┌────────────────────────────────────────────────────────────────┐
│  "Still Got Questions? We Got You"                             │
│                                                                │
│  [Categories - if 8+ questions]                                │
│  [General] [Pricing] [Technical] [Legal]                       │
│                                                                │
│  1. How much do you charge for a landing page?                 │
│     Our landing page (design + development) starts at $1800.   │
│     For Drupal only starts at $1400.                          │
│     **Want a custom quote? Book a quick call** ← Conversion!   │
│                                                                │
│  2. How long does a typical project take?                      │
│     • Simple landing page: 1-2 weeks                          │
│     • Full website: 4-6 weeks                                  │
│     • **Need it faster? Let's talk** ← Conversion!            │
│                                                                │
│  3. What happens after I pay the invoice?                      │
│     We'll schedule a kickoff call within 48 hours...          │
│                                                                │
│  [Couldn't find an answer?]                                   │
│  [Book a call] ← Leads to conversion instead of dead end       │
└────────────────────────────────────────────────────────────────┘
```

**FAQ Checklist:**
- [ ] **Specific and focused questions**: Not generic
- [ ] **Short, clear, skimmable answers**: Use bullets, bold key terms
- [ ] **Provide next steps or action links**: FAQs should lead somewhere
- [ ] **Categories**: For 8+ questions, organize them
- [ ] **Leads to conversion**: Not a dead end ("Want a custom quote? Book a call")
- [ ] **Written in customer's language**: How THEY would ask
- [ ] **Cover common objections**: Price, time, risk, complexity

**FAQ Questions Should Be:**
- ✓ Specific & Outcome focused
- ✓ Cover common objections
- ✓ Written in customer's language
- ✗ Generic ("What do you do?")

## CTA Best Practices

**Primary CTA** (above fold):
- Action verb + benefit: "Jetzt Angebot anfordern", "Kostenlos beraten lassen"
- Contrasting color
- Repeat 2-3x on page
- **Add CTA Subtext**: Reduces friction

**Secondary CTA**:
- "Mehr erfahren", "Details ansehen"
- Links to deeper content

**Form CTA**:
- Specific: "Rückruf anfordern" > "Absenden"
- Add benefit: "Unverbindlich anfragen"

**CTA Subtext Examples**:
| Primary CTA | Subtext |
|-------------|---------|
| "Start free trial" | "No credit card required" |
| "Jetzt anfragen" | "Antwort in 24 Stunden" |
| "Get started" | "Setup takes 2 minutes" |
| "Book a demo" | "30-minute call, no commitment" |

## Build in adesso CMS

Use available paragraph types. See `references/adesso-cms-components.md` for full details.

**Core Component Mapping**:

| Concept | adesso CMS Paragraph | Key Fields |
|---------|---------------------|------------|
| Hero | `hero` | field_heading, field_summary, field_link, field_link2, field_media, field_pre_headline |
| Benefits Grid | `card_group` + `card` | Cards with field_icon, field_heading, field_summary |
| Text + Image | `sidebyside` | field_heading, field_summary, field_media, field_link |
| Statistics | `statistic` + `statistic_item` | field_stat_value, field_stat_label |
| Team/Contacts | `contact_directory` + `contact_item` | field_title, field_icon, field_link (tel:) |
| Lead Form | `embed` (with Webform) | field_embed_code with webform |
| Trust Badges | `logo_collection` | field_logos, field_link |
| FAQ | `accordion` + `accordion_item` | field_heading (Q), field_body (A) |
| Downloads | `download` + `download_item` | File attachments |

**Known Gaps** (use workarounds from reference):
- ❌ No dedicated `testimonial` → Use `sidebyside` with quote styling
- ❌ No `cta_banner` → Use `hero` minimal or `sidebyside` accent theme
- ❌ No `process_steps` → Use `card_group` with numbered icons

**Implementation Order**:
1. Create/update node with basic fields (title, meta)
2. Add `hero` paragraph first (above fold) - include CTA subtext in summary
3. Add content paragraphs following the formula order
4. Configure each paragraph with content
5. Set proper headings hierarchy (H1 in hero only → H2 → H3)
6. Add internal links and CTAs (field_link, field_link2)
7. Preview with Playwright and test form submission

## Conversion Checklist

Before finalizing, verify:

### Above the Fold
- [ ] **H1 + Value prop + CTA** visible without scrolling
- [ ] **Eyebrow text** sets context for target audience
- [ ] **CTA Subtext** reduces friction ("No credit card", "Unverbindlich")
- [ ] **Product visual** shows what you do instantly

### Trust Building
- [ ] **Client logos** visible (hero or below)
- [ ] **At least 2 trust elements** (certs, testimonials, numbers)
- [ ] **Fresh testimonials** with photos, names, titles, dates
- [ ] **Stats/numbers** catch scanner attention

### Conversion Flow
- [ ] **Single focus**: One primary conversion goal per page
- [ ] **CTA repeated** 2-3x throughout page
- [ ] **F-pattern**: Important content on left side
- [ ] **Every section leads** to action or next section
- [ ] **FAQ leads to conversion** (not dead end)
- [ ] **Final CTA section** before footer

### Form Optimization
- [ ] **Max 5-7 fields**
- [ ] **Mobile-first**: Forms work on mobile, buttons thumb-friendly
- [ ] **Clear submit text**: "Rückruf anfordern" > "Absenden"
- [ ] **Success message** with next steps

### Technical
- [ ] **Mobile-first**: All elements work on mobile
- [ ] **Load speed**: Optimized images, no blocking resources
- [ ] **Contact visible**: Phone/form accessible from anywhere
