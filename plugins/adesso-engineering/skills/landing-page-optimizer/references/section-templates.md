## Hero Section

**Purpose**: Capture attention, communicate value proposition, drive first action.
**Key Insight**: You have 3-5 seconds to make a connection. 80% don't read past headline if it doesn't grab attention.

### Template Structure

```
┌────────────────────────────────────────────────────────┐
│  [Logo + Navigation]                                   │
├────────────────────────────────────────────────────────┤
│  [Eyebrow Text]           [Client Logos - Trust Bar]  │
│  "Für Landwirte in NRW"   Partner1 | Partner2 | ...   │
│                                                        │
│  H1: [3 Big Outcomes - not features]                  │
│  "Organize conversations, files, and tools"           │
│                                                        │
│  Subline: [Supporting statement - who you help,        │
│           what you do, why it matters]                 │
│                                                        │
│  [Primary CTA Button]  [CTA Subtext]                  │
│  "Jetzt anfragen"      "Unverbindlich & kostenlos"    │
│                                                        │
│  [Product Visual - Screenshot/Video/Dashboard]        │
│  Shows instantly what the product does                │
└────────────────────────────────────────────────────────┘
```

### Content Guidelines

**Eyebrow Text (NEW)**:
- Sets context, clarifies who it's for
- Examples: "Für moderne Remote-Teams", "Für Landwirte in NRW", "Enterprise-Lösung"
- Keep short: 3-5 words

**H1 Headline (max 10 words)**:
- Describe 3 big outcomes, not features
- Lead with benefit, not feature
- Be specific to the service/product
- Use power words: "Dein", "Jetzt", "Einfach", "Sicher"

**Good**: "Organize conversations, files, and tools in one place"
**Bad**: "Willkommen bei Firma XY"

**Subline (max 30 words)**:
- Expand on the headline
- Include: Who you serve + What you deliver + Key differentiator
- Can use bullet points for scanning

**CTA Button + Subtext (NEW)**:
- Action verb + Benefit
- Examples: "Jetzt Angebot anfordern", "Kostenlos beraten lassen", "Termin vereinbaren"
- **Add CTA Subtext** to reduce hesitation:

| CTA Subtext | Effect |
|-------------|--------|
| "No credit card needed" | Removes payment fear |
| "Setup takes 2 minutes" | Reduces effort perception |
| "Unverbindlich anfragen" | Removes commitment fear |
| "Kostenlose Beratung" | Adds value |
| "Antwort in 24 Stunden" | Sets expectations |

**Client Logos (NEW)**:
- Show known clients/partners to build trust instantly
- Place in hero or directly below
- 4-6 logos work best

## Problem/Solution

**Purpose**: Build empathy, show understanding of customer challenges.

### Template Structure

```
┌────────────────────────────────────────────────────────┐
│  H2: [Acknowledge the Challenge]                       │
│                                                        │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐ │
│  │ Pain Point 1 │  │ Pain Point 2 │  │ Pain Point 3 │ │
│  │ + Solution   │  │ + Solution   │  │ + Solution   │ │
│  └──────────────┘  └──────────────┘  └──────────────┘ │
│                                                        │
│  [Transition text to benefits]                         │
└────────────────────────────────────────────────────────┘
```

### Content Pattern

**Pain → Solution pairs**:

| Pain Point | Solution Framing |
|------------|------------------|
| "Unübersichtliche Prozesse" | "Transparente Digitalisierung" |
| "Unsichere Marktlage" | "Europaweites Netzwerk für Flexibilität" |
| "Schwer erreichbare Ansprechpartner" | "24/7 Erreichbarkeit" |

## Benefits Grid

**Purpose**: Highlight 3-6 key advantages with visual anchors.

### Template Structure

```
┌────────────────────────────────────────────────────────┐
│  H2: [Benefits Overview Headline]                      │
│  Subline: [Brief context]                              │
│                                                        │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐                │
│  │  Icon   │  │  Icon   │  │  Icon   │                │
│  │  Title  │  │  Title  │  │  Title  │                │
│  │  Desc   │  │  Desc   │  │  Desc   │                │
│  └─────────┘  └─────────┘  └─────────┘                │
│                                                        │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐                │
│  │  Icon   │  │  Icon   │  │  Icon   │                │
│  │  Title  │  │  Title  │  │  Title  │                │
│  │  Desc   │  │  Desc   │  │  Desc   │                │
│  └─────────┘  └─────────┘  └─────────┘                │
└────────────────────────────────────────────────────────┘
```

### Benefit Card Template

```markdown
**Icon**: [Relevant icon from library]
**Title**: [2-4 words, benefit-focused]
**Description**: [1-2 sentences explaining the benefit]
```

**Example**:
- Icon: Clock
- Title: "24/7 Erreichbarkeit"
- Desc: "Unsere Experten sind rund um die Uhr für Dich da – auch an Wochenenden und Feiertagen."

### Best Practices
- Use 3, 4, or 6 items (grid-friendly)
- Each title should be scannable standalone
- Icons should be consistent in style
- Order by importance (most important first)

## Social Proof

**Purpose**: Build trust through third-party validation.
**Key Insight**: Fresh testimonials with photos, titles, and company names add significantly more proof than anonymous quotes.

### Heading Best Practice (NEW)

**DON'T**: "Testimonials" / "Was Kunden sagen"
**DO**: "Loved by Startups Worldwide" / "Vertraut von 500+ Landwirten"

The heading should set the stage and make an impact, not just label the section.

### Elements (use 2+ of these)

**Testimonials** (Enhanced):
```
┌────────────────────────────────────────────────────────┐
│  "The old system was a mess of sticky notes and pain.  │
│   Now we can **see results within days**."             │
│   ← Bold the key outcome!                              │
│                                                        │
│  [Photo]  Sophie König                                │
│           CEO, Acme GmbH                               │
│           Juni 2024  ← Fresh dates matter!            │
└────────────────────────────────────────────────────────┘
```

**Testimonial Checklist**:
- [ ] **Profile photos**: Real faces build trust (no stock photos)
- [ ] **Bold key results**: Highlight what matters for scanners
- [ ] **Fresh dates**: Recent testimonials feel more honest
- [ ] **Full attribution**: Name + Title + Company
- [ ] **End with CTA**: "Ready to join them?" pushes action

**Numbers/Stats**:
```
┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐
│   50+    │  │  1969    │  │   24/7   │  │  100%    │
```

## Lead Form

**Purpose**: Capture lead information with minimal friction.

### Template Structure

```
┌────────────────────────────────────────────────────────┐
│  H2: [Benefit-focused headline]                        │
│  Subline: [What happens after submission]              │
│                                                        │
│  ┌──────────────────────────────────────────────────┐ │
│  │  Anrede:     [Dropdown: Frau/Herr/Divers]        │ │
│  │  Vorname:    [________________]                   │ │
│  │  Nachname:   [________________] *                 │ │
│  │  E-Mail:     [________________] *                 │ │
│  │  Telefon:    [________________]                   │ │
│  │  PLZ:        [______]                             │ │
│  │                                                   │ │
│  │  [ ] Rückruf gewünscht                           │ │
│  │  [ ] Datenschutz akzeptiert *                    │ │
│  │                                                   │ │
│  │  [    Jetzt anfragen    ]                        │ │
│  └──────────────────────────────────────────────────┘ │
└────────────────────────────────────────────────────────┘
```

### Field Guidelines

**Required fields** (mark with *):
- Name (or just Last Name)
- Email OR Phone
- Privacy consent

**Optional but useful**:
- PLZ (for regional routing)
- Callback preference
- Message/Notes

**Best Practices**:
- Max 5-7 fields
- Single column on mobile
- Inline validation
- Clear error messages
- Success message with next steps

## FAQ Section (NEW)

**Purpose**: Handle objections and drive conversion, not just answer questions.
**Key Insight**: FAQs should lead to conversion, not be a dead end.

### Template Structure

```
┌────────────────────────────────────────────────────────┐
│  H2: "Still Got Questions? We Got You"                │
│                                                        │
│  [Categories - if 8+ questions]                       │
│  [General] [Pricing] [Technical] [Legal]              │
│                                                        │
│  1. How much do you charge?                           │
│     Our service starts at €X. For custom pricing:     │
│     **Book a quick call** ← Conversion link!          │
│                                                        │
│  2. How long does it take?                            │
│     • Standard: 2-3 weeks                             │
│     • Express: 1 week (extra fee)                     │
│     **Need it faster? Let's talk** ← Conversion!      │
│                                                        │
│  [Couldn't find an answer?]                           │
│  [Book a call] ← Final conversion opportunity         │
└────────────────────────────────────────────────────────┘
```

### FAQ Best Practices

**Questions Should Be**:
- ✓ Specific & Outcome focused ("How long until I see results?")
- ✓ Cover common objections (price, time, risk, complexity)
- ✓ Written in customer's language (how THEY would ask)
- ✗ Generic ("What do you do?" "Who are you?")

**Answers Should Be**:
- Skimmable: Use bullets, bold key terms
- Include next steps or action links
- Lead to conversion, not dead end

**Heading Options**:
- "Still Got Questions? We Got You"
- "Häufig gestellte Fragen"
- "Alles was Du wissen musst"

## Final CTA Section (NEW)

**Purpose**: Last push before footer - the final conversion opportunity.
**Key Insight**: Benefit-focused subtext addresses objections (risk, effort, price).

### Template Structure

```
┌────────────────────────────────────────────────────────┐
│  H2: "Ready to simplify your workflow?"               │
│  (Strong, clear headline focusing on core outcome)    │
│                                                        │
│  Subtext: "Built for teams of any size.               │
│            Setup takes <2 minutes."                    │
│  (Addresses objections: risk, effort, price)          │
│                                                        │
│  [HIGH CONTRAST BUTTON]     [Product Image]           │
│  "Start free trial"          Makes offer feel real    │
│                                                        │
│  [Trust boosters]                                     │
│  ⭐⭐⭐⭐⭐ 4.9/5 on G2  •  "Trusted by 1000+ teams"     │
└────────────────────────────────────────────────────────┘
```

### Final CTA Checklist

- [ ] **Strong headline**: Focus on outcome ("Ready to grow faster?")
- [ ] **Benefit subtext**: Remove friction (risk, effort, price objections)
- [ ] **High contrast button**: Stands out visually
- [ ] **Trust boosters**: Ratings, reviews, client count
- [ ] **Product image**: Makes offer feel more real

### Headline Examples

| Outcome-Focused | Benefit |
|-----------------|---------|
| "Ready to simplify your workflow?" | Ease |
| "Bereit für mehr Umsatz?" | Growth |
| "Start building faster today" | Speed |
| "Join 1000+ teams already growing" | Social proof |

## Footer Section (Enhanced)

**Purpose**: Last touchpoint - navigation, trust, and contact.
**Key Insight**: Footer should re-emphasize USP with a short tagline.

### Template Structure

```
┌────────────────────────────────────────────────────────┐
│  [Clear Navigation - Mini Sitemap]                    │
│  Pages: Home | Pricing | Contact                      │
│  Legal: Privacy | Terms | Cookies                     │
│  Services: Service 1 | Service 2 | Service 3          │
│                                                        │
│  [Short Tagline - Re-emphasize USP]                  │
│  "Helping teams build faster, every single week."     │
│                                                        │
│  [Contact]              [Social Links]                │
│  support@acme.com       [Twitter] [LinkedIn]          │
│  +49 123 456 789        Shows you're active           │
│                                                        │
│  © 2024 Company. All rights reserved.                 │
└────────────────────────────────────────────────────────┘
```

### Footer Checklist

- [ ] **Clear navigation**: Mini sitemap helps users find pages
- [ ] **Short tagline**: Re-emphasize your USP (1 sentence)
- [ ] **Legal links**: Required for compliance
- [ ] **Contact links**: Build trust, direct way to reach you
- [ ] **Active social links**: Shows you're active and legitimate
